本篇题目：GPU 调度

题目

为了充分发挥 GPU 算力，
需要尽可能多的将任务交给 GPU 执行，
现在有一个任务数组，
数组元素表示在这1s内新增的任务个数，
且每秒都有新增任务，
假设 GPU 最多一次执行n个任务，
一次执行耗时1s，
在保证 GPU 不空闲的情况下，最少需要多长时间执行完成。

输入

第一个参数为 GPU 最多执行的任务个数
取值范围1 ~ 10000
第二个参数为任务数组的长度
取值范围1 ~ 10000
第三个参数为任务数组
数字范围1 ~ 10000

输出

执行完所有任务需要多少秒

示例一

输入

3
5
1 2 3 4 5

输出

6

说明

一次最多执行3个任务
最少耗时6s

示例二

输入

4
5
5 4 1 1 1

输出

5

说明

一次最多执行4个任务
最少耗时5s

代码编写逻辑

本题主要逻辑是求解任务调度的最小时间。代码首先读入任务数量 n 和任务长度 len，然后读入 len 个任务长度。接下来，代码将任务长度按照从大到小的顺序排序，然后依次执行任务。在执行任务的过程中，代码记录了当前时间和已经完成的任务长度，如果当前任务长度加上已经完成的任务长度大于 n，则需要等待一段时间才能继续执行任务。等待的时间是当前任务长度加上已经完成的任务长度减去 n。如果当前任务长度加上已经完成的任务长度小于等于 n，则可以直接执行任务。当所有任务都执行完毕后，如果还有任务没有完成，则需要继续等待，直到所有任务都完成。最后，代码输出完成所有任务所需的最小时间。

Code

import java.util.Arrays;
import java.util.Comparator;
import java.util.Scanner;

public class Main {
  public static void main(String[] args) {
    Scanner s = new Scanner(System.in);
    int n = s.nextInt();
    int len = s.nextInt();
    Integer[] jobs = new Integer[len];
    for (int i = 0; i < len; ++i) {
      jobs[i] = s.nextInt();
    }

    Arrays.sort(jobs, Comparator.reverseOrder());

    int time = 0;
    int more = 0;
    for (int i : jobs) {
      more = (i + more > n) ? i + more - n : 0;
      time++;
    }
    while (more > 0) {
      more -= n;
      time++;
    }

    System.out.print(time);
  }
}
